# Wenai Listing Factory RC 截图验收清单

本清单用于 RC Freeze 后的人工截图验收。当前项目未新增 Playwright 或截图依赖；如当前环境无法自动截图，需要人工分别打开桌面端和移动端视口完成确认。

## 验收方式

1. 启动本地预览，建议地址为 `http://127.0.0.1:3000`。
2. 依次打开下方页面。
3. 每页截取桌面端首屏和移动端首屏。
4. 按页面检查项逐项勾选，发现问题只做 RC 级修复：命名污染、乱码、错误链接、重复 demo boundary、明显数据口径不一致。

## 通用检查项

每个页面都需要确认：

- 桌面端首屏是否能看懂当前页面的作用。
- 移动端是否没有整页横向溢出。
- 主 CTA 是否唯一且明确。
- 上一站 / 下一站是否符合演示主线。
- DemoBoundaryNotice 是否没有重复出现。
- 是否没有乱码。
- 是否没有不必要的英文残留。
- 是否没有内部 key 或 mock 字段裸露。
- 是否没有过度承诺，例如自动保证爆款、自动投放、真实平台回流、已保存到客户系统。
- 是否能自然进入下一步演示。

## 页面清单

### `/`

- 桌面端：首屏应明确表达 Wenai 是电商上新内容生产线，并能看到开始公开试跑或 5 分钟演示路径入口。
- 移动端：Executive Demo Mode 不应挤压主 CTA，横向 pill 如存在可滚动但整页不能溢出。
- 下一步：应能进入 `/insights`、`/factory` 或 `/poc`。

### `/factory`

- 桌面端：内容工厂控制台、KPI、最近操作流、任务队列和本周生产复盘应层级清楚。
- 移动端：KPI 和任务卡应自然堆叠，不出现表格撑宽页面。
- 下一步：应能进入 `/calendar` 或 `/poc/report`。

### `/insights`

- 桌面端：类目洞察库应突出可复用结构和生成 Brief 的操作。
- 移动端：洞察卡片不应过密，按钮不应换行挤压。
- 下一步：生成 Brief 草稿后应能进入 `/briefs`。

### `/briefs`

- 桌面端：Brief 资产库应能清楚看到筛选、Brief 状态和加入任务 / 报告 / 交付包操作。
- 移动端：筛选项允许横向滚动，但卡片内容不能溢出屏幕。
- 下一步：应能进入 `/factory` 或 `/calendar`。

### `/calendar`

- 桌面端：未来 7 天内容计划应像轻量生产排期，不像复杂日历软件。
- 移动端：日期卡片应纵向堆叠，状态操作不能挤压。
- 下一步：应能进入 `/poc/report`。

### `/pipelines/new-listing`

- 桌面端：新品上新流水线应显示 SKU、类目规则、品牌禁区、类目灵感和批量 Brief。
- 移动端：流程和 Brief 卡片不应横向撑开页面。
- 下一步：应能进入 `/poc/report`。

### `/poc`

- 桌面端：公开试跑配置器应清楚说明选择 SKU、平台、内容目标和输出范围。
- 移动端：配置项应易读，主 CTA 不被二级入口抢走。
- 下一步：应能进入 `/poc/report`。

### `/poc/report`

- 桌面端：POC 试跑交付报告应适合截图给客户或老板，客户交付包区域应清楚。
- 移动端：报告段落不应过长挤在一起，CTA 应清晰。
- 下一步：应能进入 `/review` 或 `/inquire`。

### `/review`

- 桌面端：POC 复盘看板应突出 Brief 质量评分、风险复盘和下一步生产建议。
- 移动端：评分卡和风险项应可读，不出现横向溢出。
- 下一步：应能进入 `/clients` 或 `/inquire`。

### `/clients`

- 桌面端：客户项目空间应能按项目看到 Brief、报告、交付包、风险和销售话术。
- 移动端：展开区域不应太密，项目卡之间间距稳定。
- 下一步：应能进入 `/pricing` 或 `/admin/inquiries`。

### `/inquire`

- 桌面端：页面应承接带报告咨询，而不是普通 Contact Us。
- 移动端：试跑摘要和表单字段应分组清楚。
- 下一步：应能进入 `/pricing`。

### `/pricing`

- 桌面端：根据本次试跑推荐 Growth 的逻辑应清楚，套餐不只是功能列表。
- 移动端：套餐卡片应纵向排列，主咨询 CTA 不应被挤压。
- 下一步：应能进入 `/admin/inquiries`。

### `/admin/inquiries`

- 桌面端：商务推进后台应能看到来源路径、复盘结论、风险、评分、套餐和下一步动作。
- 移动端：如存在表格，必须在容器内滚动或转为卡片，不允许整页横向溢出。
- 下一步：应能推进商机阶段或结束演示。

## 截图交付建议

- 桌面端宽度建议：`1440px`。
- 移动端宽度建议：`390px` 或常见 iPhone 宽度。
- 文件命名建议：`listing-factory-rc-{route}-{desktop|mobile}.png`。
- 截图后优先检查主线是否顺畅，再检查局部视觉 polish。
