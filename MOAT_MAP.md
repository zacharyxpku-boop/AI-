# Wenai · Moat Map(2026-04-26)

阁主裁决稿。这份文档回答两个问题:**wenai 比谁强、强在哪、对手多久能抄。**

格式致敬 clico-clean/MOAT_MAP.md,但 wenai 的护城河结构跟 clico 完全不同——
clico 是"单点 SaaS 比 Arcads 快",wenai 现在收敛为"**SKU 上新物料包子站**":不卖 19 个工具,先把一个 SKU 从信息输入跑到可验收交付物。

---

## 1. 一句话杀器

> **「跨境电商商家来一次,主图 → 模特图 → 详情页 → 短视频 → 客服话术 → 询盘对接 全套交付完。」**
>
> 不是单点工具,是商家在 wenai 里跑完一个 SKU 从空白到上架的完整流水线。

副标题:
- **不卖功能,卖交付物**:上新 SOP、主图方向、详情页文案、合规提醒、客服话术、复评 checklist。
- **不在子站收款**:演示看形态,10 SKU POC 看真实价值,正式支付和合同走独立主站。

**Wenai 独占的四件事**(其它家 6+ 个月内复制不了):

### MOAT-01 · 爆款视频拆解 → 静态图 prompt 直通车
任何家都没有。Picset 有"爆款链接复刻"但只针对静态详情页,不能拆视频。clico 做了视频拆但只用于自家视频生成,没接到静态图工厂。
**wenai 的杀手链路:扔一个 TikTok 爆款 → Gemini 拆出 6 个镜头各自的图像 prompt → 每个 prompt 一键带去 AI 影棚生你的产品同款图。**这是单工具竞品永远做不到的——他们没有 AI 影棚这个下游配套。

### MOAT-02 · 反向意图扩客
阿里万相 LMA3 同款方法论(美甲师→电子锁),但万相只在淘内,wenai 是产品给跨境/独立站商家的可购买工具。给商家的是 5-8 个非显然客群 + 卖点翻译 + 触达渠道 + 真实搜索词。竞品全在做"产品→客户标签",没人做反向。

### MOAT-03 · 全链路工厂(8 photoshoot + 视频 + 拆解 + 文案 + 客服 + 询盘)
所有竞品都是单点工具——"我会给你做模特图"、"我会给你做视频"。wenai 是**单一登录 + 同一品类配置 + 跨模块复用**:你在影棚生成的模特图,右键复制 URL 直接喂给 AI 视频;视频拆解出的 prompt 直接带去影棚;反向意图挖出的人群标签直接传给文案模块。**别人是离散工具集合,wenai 是连贯流水线。**

### MOAT-04 · ToB 询盘 + 私域微信全栈
跨境老板要的不是"我能用",是"我能找谁问、谁能给我定制、谁能把我的供应链对接进来"。wenai 有 `/inquire` 自助提交 + admin 收件箱 + 微信渠道,clico 跟 Picset 都没做。这块不是技术护城河,是**关系密度护城河**——一旦有 30 个真实跨境老板进来,他们会互相推荐,后来者要追同样规模的客户密度需要 6-12 个月。

### MOAT-05 · SKU 全生命周期闭环 (2026-04-27 新增)
单点工具的最大问题: 客户做完一个动作就走,没下一步指引,留存差。
wenai 现在有完整闭环: **选品(#1) → 反向意图 → 拆解 → 影棚 → 视频 → 测款(#11) → 上新 → 数据洞察(#10) → 回选品**。每个模块的"下一步"都明确指向闭环里的下个节点(`NextStepHint` 组件),客户跑完一个 SKU 自动进入下个 SKU。
**这是 retention 护城河**: 商家用得越深, 历史数据越多, 越离不开。竞品做单点工具的根本搭不出这个循环。clico 也只做"视频复刻"单环,没有商业闭环。

### MOAT-06 · 决策层 vs 执行层 二元产品架构
所有 AI 工具都集中在"执行层"(给我生张图 / 写段文案), 没有"决策层"(我该选哪个 SKU / 该投哪张图 / 数据该怎么解读)。
wenai 把决策层(选品 / 测款 / 数据洞察)和执行层(影棚 / 视频 / 拆解 / 文案)显式分开,UI 颜色编码区分(决策蓝 cat-content / 执行金 accent)。**决策层模块是商家"动脑"的地方,粘性极强,因为商家一旦把品类配置/历史数据交给 wenai,迁移成本就指数上升。**

### MOAT-07 · 工厂入口 (batch-launch)
真正把 11 pipelines 串成一条流水线的是 `/pipelines/batch-launch`。商家贴 5-20 个 SKU,选 6 工序,系统一次生成完整 SOP — 含每个 SKU 在每个工序的具体 prompt + 参数 + 验收标准。
**这是从 SaaS 工具进化为"数字员工" 的关键一步**: 商家不再"我去 wenai 跑一个工具",而是"我把这批 SKU 交给 wenai,自己去做更高价值的事"。竞品没有这种"批量编排"维度的玩家。

## 3. Moat 半衰期估算

| Moat | 复制窗口 | 决定半衰期的关键 |
|---|---|---|
| MOAT-01 视频拆解→静态图直通 | **9-12 个月** | 单点工具复制单点功能容易,但要把 8 个 photoshoot 模式 + 视频 + 拆解串成同一个 SKU 配置才有意义。竞品每加一个工位就拉远 |
| MOAT-02 反向意图扩客 | **6 个月** | prompt engineering 不是壁垒,但需要 DeepSeek/qwen 长期调教 + 真实电商案例反馈。竞品照抄 prompt 一周能上,但效果没法跟用户回流数据飞轮比 |
| MOAT-03 全链路工厂 | **12-18 个月** | 这是产品架构护城河。竞品要从单工具变工厂,等于重写整个产品。商业上他们也不愿意——破坏自己的单工具定价模型 |
| MOAT-04 ToB 询盘 + 私域 | **3-6 个月** | 技术护城河弱,但**关系密度**护城河强。每多一个客户,新进入者越难追 |
| MOAT-05 SKU 全生命周期闭环 | **18-24 个月** | retention 飞轮,客户用得越深越锁定。竞品要重写产品哲学。这是 wenai 最深的护城河 |
| MOAT-06 决策-执行二元架构 | **12 个月** | 决策层模块(选品/测款/数据洞察)单独抄不难,但抄完没用 — 商家不会单买决策工具,要跟执行层一起买 |
| MOAT-07 工厂入口(batch-launch) | **9 个月** | 把多模块串成一条 SOP 的产品架构,本身不复杂,但需要其他 10 模块齐备才有意义 |
| 中国 AIGC 合规栈 | **12 个月** | 海外竞品没动力。GB/T 42888 + 深度合成规定 + 平台规则版本管理是时间成本 |

## 5. 不做什么(资源约束)

- ❌ **重型异步队列(BullMQ + Redis worker)**:clico 有,wenai 现在不需要——视频生成 60-90s 同步等够用,客户量上来再上
- ❌ **Supabase Postgres**:Redis 够,过早优化是创业大忌
- ❌ **Sora API / Runway / Kling**:wanx i2v 性价比已经够,加贵的视频模型只是给"高端版"加价用,等 ToB 客户付钱再说
- ❌ **第四第五家生图(fal.ai / Replicate)**:OpenAI gpt-image-1 + wanx 双轨够用,加 N 家维护成本爆
- ❌ **自己训模型 / fine-tune**:护城河不在模型,在工厂

## 6. 下一轮竞品防守动作(优先级)

1. **MOAT-01 闭环要做实** — video-teardown 的 prompt 现在能拷到 AI 影棚,但需要做一键自动跳转 + 自动填表
2. **MOAT-02 prompt 持续调教** — 收集 50+ 真实电商案例反向验证 LMA3 同款方法论的输出质量
3. **MOAT-03 sidebar 配置面板** — 让客户进来选"我是 [服装/家居/汽配/数码/工具]",所有 8+ 模式自动按品类预设 prompt(category-prompts.ts 已有底子,UI 没暴露)
4. **MOAT-04 admin/inquiries 自动派单** — 询盘进来后自动发邮件给作者 + 飞书机器人,响应时间从 24h 压到 4h
