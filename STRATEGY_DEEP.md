## L1 · 商业故事(一句话杀器 + 商业模型)

### 故事的三圈层

```
最里圈(产品定位)         "中国跨境/本土电商工厂级 AI SaaS"
                                 ↑
中圈(商业模式)        "按 SKU 包月,从美工 / 摄影 / 文案 替代 → 数据驱动决策"
                                 ↑
外圈(行业意义)    "OPC(One-Person Company)电商 OS,让一人团队跑出十人产能"
```

### 三句话给不同受众

| 听众 | 一句话 |
|---|---|
| 跨境老板 | "少一个美工 + 少 0.5 个摄影 + 少 0.5 个运营,每月省 ¥10-30K 人力,wenai 自助跑" |
| 媒体/分析师 | "Picset AI × 美图设计室 × 阿里万相 LMA3 × clico 拆解能力,合并到一个 ToB 工厂" |
| 投资人 | "电商工厂级 AI 工具的 Shopify Plus,从 SaaS 工具 → 数字员工的窗口期" |
| 同行 | "我们卖的不是工具,是商家从空白 SKU 到上架的完整流水线" |

### 商业模型(三档定价的内生逻辑)

```
演示                 子站免费   看输出形态,不承诺真实 SKU 效果
10 SKU POC           主站支付   验证上新物料包、复改边界、复评节奏
Enterprise           合同定价   API / ERP / 独立站嵌入 / SLA / 发票
```

不靠"免费试用 → 自动转化"那套,因为当前代码会作为独立主站里的子站。
靠"演示 SKU → 接入需求 → 10 SKU POC → 主站支付/合同"形成可控漏斗。

## L2 · 用户分层 ICP(Ideal Customer Profile)

### 三档用户的痛点完全不同

#### A. 中小卖家(年营收 < 1 亿)
- **核心痛点**: 没团队,自己又要选品又要做图,精力杠杆为零
- **wenai 给什么**: AI 影棚 + AI 视频 + 翻译 + 文案,**一个人撑 5 个人产能**
- **付费意愿**: ¥299/月 起,但要看到第一张图就走
- **决策周期**: 即时(刷视频 → 进站 → 试 → 买)
- **现在 wenai 做到 80%**: 影棚生图够了,但首页落地页还没把这群人"惊艳"住

#### B. 中腰部品牌(年营收 1-10 亿)
- **核心痛点**: 多平台运营顾此失彼,每月烧广告 ROI 看不见,内容生产瓶颈
- **wenai 给什么**: 全套 7 pipelines + 平台调性适配 + 数据洞察
- **付费意愿**: ¥1499-9999/月,要看到"组织效率提升"
- **决策周期**: 2-4 周(先内部试 → POC → 决策)
- **现在 wenai 做到 50%**: 工具有了,但**多 SKU 批量** + **品牌语料库** + **跨渠道效果归因** 全没

#### C. 头部品牌(年营收 > 10 亿)
- **核心痛点**: 跨部门 AI 能力散落,数据飞轮没建起来,IT 决策慢
- **wenai 给什么**: 私有化部署 + DPA + Bot 集成 + 战略咨询(询盘渠道)
- **付费意愿**: 年合同 ¥30-100 万,要看到 ROI 报告
- **决策周期**: 3-6 个月(SaaS 选型 → 法务 → 试点 → 全量)
- **现在 wenai 做到 20%**: 询盘 + DPA 模板有了,**但私有化部署 / 私有模型微调 / SLA 保障** 全没

**关键判断**: 接下来 6 个月**主攻 A + B**,C 只接锚点客户(wzq 朋友 1 个),不烧人力做 enterprise 销售。

## L3 · 技术壁垒(三层堆叠)

clico/MOAT_MAP 用的是"可复制窗口期"角度。这里换个角度:
**模型层 / 数据层 / 上下文层** 三层堆叠,越下层越难复制。

```
┌─────────────────────────────────────────────┐
│  模型层 (3 个月就被超越)                      │
│  · gpt-image-1 / wanx i2v / Gemini Vision   │
│  · 我们调谁的 API,别人也能调                  │
│  · 不是壁垒,是基础设施                       │
├─────────────────────────────────────────────┤
│  数据层 (12 个月才能积累)                     │
│  · ecom-prompts.ts 13 品类 × 12 场景 × 8 风格 │
│  · category-prompts.ts 5 大品类 prompt 调教  │
│  · 真实电商案例的 prompt 反向迭代             │
│  · 平台规则数据库 (AIGC 合规 / 商标黑名单)    │
│  · 这是 wenai 真正的数据资产                  │
├─────────────────────────────────────────────┤
│  上下文层 (24 个月才能形成)                   │
│  · 商家 SKU 历史 + 偏好 + 反馈                │
│  · "客户 A 的产品语境" 沉淀越深越锁定         │
│  · 跨模块复用: 影棚生的图直接喂视频/文案/外联 │
│  · 关系密度: 用户互相推荐 + ToB 定制          │
│  · 这是终极护城河                             │
└─────────────────────────────────────────────┘
```

**当前 wenai 的现状**:
- 模型层 ✅ 接齐(HappyHorse + Gemini + DashScope)
- 数据层 🟡 已有 ecom-prompts / category-prompts 底子,但**没建反馈飞轮**(用户用了哪个 prompt 出图好/不好,数据没回流)
- 上下文层 ❌ **完全没做**(商家配置不持久化 / 跨模块上下文不共享)

**P0 杀手锏 = 把上下文层激活**:商家进 wenai 第一次填"我是 [服装] 卖家,主营 [女装短裙] 价位 [¥99-299]",所有模块自动按这套上下文调 prompt + 配额,再也不重复填。

## L4 · 真实电商痛点矩阵(SKU 全生命周期)

不按"功能" 切,按"商家从看到一个市场机会到上架成单"完整 9 步:

| # | 痛点 | 商家的真实场景 | wenai 现在 | 缺口 |
|---|---|---|---|---|
| 1 | **选品** | 拍脑袋 / 看销量榜 / 抄爆款 | ❌ 没做(intent-mining 是反向客群,不是选品) | 接 1688/淘宝 / Amazon BSR API → AI 推荐 SKU |
| 2 | **品类卖点提炼** | 自己写不出戳人的卖点 | ✅ 反向意图扩客 (MOAT-02) | 已 ship |
| 3 | **主图详情** | 找美工反复改 | ✅ AI 影棚 8 模式 + 行业模板 | 已 ship |
| 4 | **模特/真人摄影** | ¥3-8K/组,慢且不可复用 | ✅ outfit-swap / model-on / pose-change | 已 ship |
| 5 | **短视频** | ¥500-3K/条,改一处全重拍 | ✅ AI 视频 hh + video-teardown 拆解 | 已 ship |
| 6 | **多语言** | 跨境必须,人工翻译贵且慢 | ✅ translate toolbox | 待加强 (按品类术语库) |
| 7 | **平台审美适配** | 每个平台审美不同要重做 | ✅ platform-style 4 平台 | 已 ship |
| 8 | **达人/KOL** | 一周发 50 封达人邮件回复率 < 3% | ✅ influencer-outbound | 已 ship |
| 9 | **客服 7×24** | 半夜没人回复就丢单 | 🟡 customer-service toolbox 弱 | 接电商平台 webhook + AI 自动回复 |
| 10 | **数据洞察 / 选品反馈** | 卖了两周不知道为啥下滑 | ❌ data-insights 弱 | **P0 缺口**:接 GA / 平台后台 + AI 解读 |
| 11 | **测款 / A-B** | 投不出爆款 → 无数据 → 不敢加预算 | ❌ 没做 | 一图变 9 张测点击率 |
| 12 | **库存/履约** | 单日多平台对账难 | 不做(超出 wenai 边界) | 留给 ERP |

**判断**:
- **痛点 1 (选品)** 和 **痛点 10 (数据洞察)** 是 wenai 现在最大缺口 — 这两个不做,wenai 永远是"内容工厂"不是"经营大脑"
- 痛点 9 (客服) 半成品,要么干脆别提要么深做
- 痛点 11 (测款) 是搭桥功能 — 把现有影棚 8 模式串成"出 9 张图 → 三平台广告投放 → 数据回流"闭环

### 新出现的风险(本轮 11 pipelines + 工厂入口落地后)

6. **决策层 ≠ 真数据飞轮**: MOAT-06 强调决策层粘性, 但 wenai 还没沉淀一条用户的"历史 SKU+反馈"数据. 商家关掉浏览器 → 状态全丢. **必须做用户级别的 SKU 历史持久化** 否则决策层是空壳
7. **batch-launch 上限 20 SKU**: 真做工厂的客户一个月 50-200 SKU. 现在 20 是 token 上限, 该上分批 + 异步队列了 (clico 的 BullMQ 该搬)
8. **STRATEGY_DEEP / MOAT_MAP 写得越完整越像 PPT**: 战略文档不卖货, 真客户演示需要 60 秒视频或 LiveDemo 录屏, 我们一份都没有
9. **AI 模型成本估算偏低**: cost-cap.ts 里的估算是"理论值", 实际 HappyHorse 真消耗未跟踪. 一个高消耗用户单日可能爆 ¥500+ 我们浑然不知
10. **没有"客户成功"反馈**: wenai 现在所有改进都是阁主一拍脑袋. 没有真实 5 个客户的"我用 wenai 这个月赚了多少 / 省了多少" 案例数据回流
